Resilient Distributed Datasets (RDDs)

  • Descripción
  • Contenidos
  • Conclusión

DESCRIPCIÓN

Un RDD es una colección de datos distribuida, inmutable y dividida en particiones. Fue la primera abstracción principal de Spark y ayuda a entender cómo se reparte el trabajo entre executors.

En proyectos actuales se suelen preferir DataFrames, porque tienen esquema y Spark puede optimizarlos automáticamente. Aun así, los RDD son útiles para aprender el modelo interno de Spark y para transformaciones de bajo nivel.

CONTENIDOS

Qué hay que recordar de un RDD

  • Distribuido: los datos se dividen en particiones.
  • Inmutable: cada transformación crea un nuevo RDD.
  • Tolerante a fallos: Spark puede reconstruir particiones mediante el linaje.
  • Evaluación perezosa: las transformaciones no se ejecutan hasta llamar a una acción.

Transformaciones y acciones

Las transformaciones crean otro RDD. Las acciones disparan la ejecución y devuelven un resultado al driver o escriben datos fuera de Spark.

Tipo Ejemplos Qué hacen
Transformaciones map, filter, flatMap, reduceByKey Preparan un nuevo RDD, pero no ejecutan todavía el cálculo.
Acciones count, collect, take, first, reduce Ejecutan el plan y devuelven resultados.

Ejemplo mínimo

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("RDD_basico").master("local[*]").getOrCreate()
sc = spark.sparkContext

rdd = sc.parallelize(["spark", "hadoop", "spark", "kafka"], 2)
conteo = rdd.map(lambda palabra: (palabra, 1)).reduceByKey(lambda a, b: a + b)

print(conteo.collect())
spark.stop()

En este ejemplo, map y reduceByKey son transformaciones. La ejecución real se produce cuando se llama a collect().

Particiones y persistencia

Las particiones determinan cuántas tareas puede ejecutar Spark en paralelo. Si hay muy pocas particiones, no se aprovechan todos los cores; si hay demasiadas, aumenta la sobrecarga de planificación. 

rdd = sc.parallelize(range(100), 4)
print("Particiones:", rdd.getNumPartitions())

Si un RDD se reutiliza varias veces, puede persistirse en memoria para evitar recalcularlo. 

rdd.persist()
print(rdd.count())
print(rdd.sum())

Cuándo usar RDD y cuándo DataFrame

Usa RDD cuando... Usa DataFrame cuando...
Necesitas control de bajo nivel sobre cada elemento. Trabajas con datos tabulares o con esquema.
Quieres explicar particiones, linaje y transformaciones básicas. Quieres que Spark optimice el plan automáticamente.
La lógica no encaja bien en SQL o DataFrames. Necesitas menos código y mejor rendimiento general.

CONCLUSIÓN

Los RDD son importantes para entender Spark, pero no deberían ser la primera opción para todos los problemas. En aplicaciones modernas, empieza con DataFrames y usa RDD cuando necesites control fino o quieras trabajar a bajo nivel.
