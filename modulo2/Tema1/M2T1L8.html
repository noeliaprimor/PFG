DESCRIPCIÓN

Esta lección recopila recursos para reforzar los contenidos del Tema 1: arquitectura de Apache Spark, componentes de la pila unificada, gestores de clúster, RDD, instalación, configuración y ejecución de las shells de Spark.

La selección está orientada a consolidar una base técnica fiable antes de trabajar con APIs avanzadas, librerías de Spark, optimización y procesamiento en tiempo real.

CONTENIDO

Documentación oficial imprescindible

  • Apache Spark Documentation: punto de entrada a la documentación oficial, instalación, ejecución, APIs y componentes.
  • Spark Quick Start: primera guía práctica para ejecutar Spark localmente, crear datasets y lanzar acciones.
  • Cluster Mode Overview: explicación de driver, executors, cluster manager y flujo de ejecución de una aplicación.
  • Submitting Applications: referencia para ejecutar jobs con spark-submit y configurar dependencias.

Gestores de clúster

  • Spark Standalone Mode: despliegue propio de Spark, útil para laboratorios y clústeres sencillos.
  • Running Spark on YARN: integración con Hadoop YARN, habitual en plataformas Big Data tradicionales.
  • Running Spark on Kubernetes: ejecución de aplicaciones Spark en contenedores, con pods para driver y executors.

RDD y modelo de ejecución

  • RDD Programming Guide: referencia para comprender particiones, transformaciones, acciones, persistencia y tolerancia a fallos.
  • Spark research paper: artículo original que ayuda a entender el diseño de Spark y el procesamiento en memoria.
  • Spark Tuning: introducción a memoria, serialización, particionado y coste de operaciones distribuidas.

Entornos locales y aprendizaje práctico

  • Docker Compose Documentation: útil para levantar laboratorios reproducibles con master, workers y servicios auxiliares.
  • Spark Packages: catálogo de conectores y extensiones para ampliar aplicaciones Spark.
  • Apache Spark GitHub: código fuente, ejemplos, issues y evolución del proyecto.
  • Databricks: What is Apache Spark?: explicación conceptual del ecosistema Spark y sus casos de uso.

RUTA RECOMENDADA

  1. Revisar Quick Start para ejecutar Spark localmente.
  2. Leer Cluster Mode Overview para diferenciar driver, executors y cluster manager.
  3. Practicar con spark-submit y shells de PySpark o Scala.
  4. Profundizar en RDD para entender particiones, transformaciones perezosas y acciones.
  5. Comparar Standalone, YARN y Kubernetes en función del entorno de despliegue.

EJERCICIOS PROPUESTOS

  1. Ejecuta un ejemplo con spark-submit y documenta driver, executors y stages en la Spark UI.
  2. Escala un entorno Standalone a varios workers y compara la distribución de tareas.
  3. Reduce memoria de executor y analiza el efecto sobre tiempo, spills o errores.
  4. Consulta logs del master y workers para localizar inicio, asignación de recursos y finalización del job.
  5. Elabora una tabla comparando Standalone, YARN y Kubernetes para un laboratorio docente.
