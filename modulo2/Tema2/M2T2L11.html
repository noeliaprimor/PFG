DESCRIPCIÓN

Esta lección reúne recursos para profundizar en el desarrollo de aplicaciones Spark: creación de SparkSession, carga y guardado de datos, RDD, DataFrames, Datasets, particionado, transformaciones, acciones y optimización de consultas.

Los enlaces están seleccionados para acompañar el ejercicio final del tema, donde se espera razonar sobre la elección de APIs, formatos de datos, paralelismo y rendimiento.

CONTENIDO

APIs principales de Spark

  • RDD Programming Guide: transformaciones, acciones, particiones, persistencia y variables compartidas.
  • Spark SQL, DataFrames and Datasets Guide: referencia central para trabajar con datos estructurados.
  • PySpark API Reference: documentación de clases y funciones usadas en aplicaciones Python.
  • Spark SQL Scala API: referencia útil cuando se trabaja con Dataset tipado en Scala.

Carga, guardado y formatos

  • Generic Load/Save Functions: lectura y escritura con DataFrameReader y DataFrameWriter.
  • Parquet Data Source: formato columnar recomendado para analítica por rendimiento y compresión.
  • JSON Data Source: lectura de datos semiestructurados, inferencia de esquema y opciones de parseo.
  • JDBC To Other Databases: integración con bases relacionales, particionado de lectura y escritura.

Particionado y optimización

  • Spark Tuning: memoria, serialización, caché, shuffle y recomendaciones generales de rendimiento.
  • Spark SQL Performance Tuning: particionado, broadcast joins, caché, Adaptive Query Execution y configuración SQL.
  • Spark Configuration: parámetros de ejecución, memoria, cores, shuffle y propiedades de aplicación.

Bibliografía y aprendizaje complementario

  • Learning Spark, 2nd Edition: libro práctico sobre DataFrames, Spark SQL, streaming y MLlib.
  • Designing Data-Intensive Applications: referencia para entender sistemas distribuidos, almacenamiento y procesamiento.
  • Spark Examples: ejemplos oficiales del repositorio de Apache Spark.
  • Spark Packages: catálogo de conectores y librerías externas para aplicaciones Spark.

RUTA RECOMENDADA

  1. Repasar RDD para comprender el modelo distribuido y la evaluación perezosa.
  2. Pasar a DataFrames y Spark SQL para trabajar con optimización Catalyst.
  3. Practicar lectura y escritura en CSV, JSON, Parquet y JDBC.
  4. Medir el impacto de repartition, coalesce, caché y particionado físico.
  5. Usar la Spark UI para interpretar stages, tasks, shuffle y tiempos de ejecución.

APLICACIÓN AL EJERCICIO

Para el ejercicio final del tema, usa estos recursos para justificar:

  • Qué API eliges para cada operación: RDD, DataFrame o Dataset.
  • Qué formato de almacenamiento conviene usar y por qué.
  • Cómo particionas datos de entrada y salida.
  • Qué transformaciones provocan shuffle y cómo reducir su coste.
  • Qué evidencias de rendimiento aporta la Spark UI.
