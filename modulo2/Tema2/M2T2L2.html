DESCRIPCIÓN

En esta lección veremos cómo diseñar, probar y desplegar una aplicación Spark, desde el desarrollo local hasta su ejecución en un clúster, con recomendaciones sobre empaquetado, dependencias y buenas prácticas.

ESTRUCTURA DEL PROYECTO

Python (PySpark)

myapp/
  app/
    jobs.py
  requirements.txt
  Dockerfile
  run.sh

Incluye un script run.sh para ejecutar en distintos entornos y un requirements.txt para dependencias.

Scala (sbt)

myapp/
  build.sbt
  src/main/scala/com/example/MyApp.scala
  src/main/resources/

EJECUCIÓN EN LOCAL Y CLÚSTER

Modo local (PySpark)

pyspark --master local[*]
python app/jobs.py --input data/input.csv --output out/

Uso de spark-submit

/opt/spark/bin/spark-submit \
  --master spark://master:7077 \
  --deploy-mode cluster \
  --num-executors 4 \
  --executor-memory 2G \
  app/jobs.py

Depuración y logs

Usa el UI (por defecto 4040) y configura niveles de log a WARN/INFO en log4j.properties.
