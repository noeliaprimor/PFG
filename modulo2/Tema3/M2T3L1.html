DESCRIPCIÓN

Apache Spark ofrece una pila de librerías que extienden su motor central (Spark Core) para cubrir casos de uso concretos: Spark SQL para datos estructurados, MLlib para machine learning, GraphX para grafos y Structured Streaming para flujos. Además existen paquetes externos y conectores (JDBC, Kafka, Cassandra, Parquet, etc.)

Capítulo X: Librerías disponibles en Apache Spark

1. Introducción

Apache Spark es un framework unificado para procesamiento distribuido que integra múltiples librerías especializadas dentro de un mismo ecosistema. Estas librerías permiten abordar diferentes tipos de cargas de trabajo, incluyendo procesamiento batch, streaming, machine learning, análisis de grafos y consultas SQL, además de facilitar la integración con múltiples fuentes de datos.

Una de las principales ventajas de Spark es que todas estas librerías comparten el mismo motor de ejecución, el mismo planificador y un modelo común de despliegue sobre distintos cluster managers, permitiendo construir pipelines completos sin cambiar de tecnología.

2. Spark Core

2.1 Descripción

Spark Core es la base del sistema. Proporciona las capacidades fundamentales de ejecución distribuida y tolerancia a fallos, sobre las cuales se construyen el resto de módulos del ecosistema.

2.2 Funcionalidades principales

  • API basada en RDD (Resilient Distributed Datasets).
  • Gestión del paralelismo y particiones.
  • Planificación de tareas (DAG scheduler y task scheduler).
  • Gestión de memoria, caché y persistencia.
  • Tolerancia a fallos mediante recomputación basada en linaje (lineage).
  • Soporte para ejecución en distintos cluster managers (Standalone, YARN, Kubernetes, entre otros).

3. Spark SQL

3.1 Descripción

Spark SQL es el módulo encargado del procesamiento estructurado. Introduce el modelo basado en DataFrames y Datasets e integra un motor SQL que permite ejecutar consultas declarativas sobre datos distribuidos.

3.2 Capacidades

  • Consultas SQL estándar sobre datos estructurados.
  • Uso de DataFrames y Datasets como abstracciones principales.
  • Integración con Hive (metastore, tablas y formatos).
  • Lectura y escritura de múltiples formatos (Parquet, ORC, JSON, CSV, Avro mediante dependencias, etc.).

3.3 Componentes clave

  • Catalyst Optimizer: optimización de planes lógicos.
  • Tungsten: mejoras de ejecución y gestión eficiente de memoria.
  • Planificación lógica y física: selección de operadores y estrategias de ejecución.

3.4 Ejemplo

df.createOrReplaceTempView("empleados")
spark.sql("SELECT departamento, COUNT(*) FROM empleados GROUP BY departamento")

4. Spark Structured Streaming

4.1 Descripción

Spark Structured Streaming permite el procesamiento de datos en tiempo real utilizando el mismo API estructurado que en batch (DataFrames/Datasets). Su modelo se basa principalmente en micro-batches, aunque se apoya en planificación incremental y gestión de estado para proporcionar consistencia.

4.2 Características

  • API unificada batch/streaming: la misma lógica puede aplicarse a datos históricos y a datos en tiempo real.
  • Soporte de fuentes y sinks: Kafka, ficheros, sockets, entre otros (dependiendo de conectores).
  • Gestión de estado para operaciones con agregaciones y ventanas.
  • Garantías de consistencia (dependen del origen/sink y del modo de escritura).

4.3 Ejemplo básico (conceptual)

stream_df = spark.readStream.format("kafka").option("subscribe", "topic").load()

query = (stream_df
    .writeStream
    .format("console")
    .start())

5. MLlib (Machine Learning Library)

5.1 Descripción

MLlib es la librería de aprendizaje automático de Spark. Está orientada a construir pipelines de ML escalables sobre grandes volúmenes de datos distribuidos, integrándose de forma natural con DataFrames mediante la API de pipelines.

5.2 Capacidades

  • Clasificación (por ejemplo, regresión logística, árboles, etc.).
  • Regresión (lineal, regularizada, etc.).
  • Clustering (por ejemplo, K-Means).
  • Reducción de dimensionalidad y extracción de características.
  • Evaluación y validación cruzada.
  • Pipelines de ML (estimators, transformers y evaluadores).

5.3 Ejemplo simple

from pyspark.ml.classification import LogisticRegression

lr = LogisticRegression(featuresCol="features", labelCol="label")
model = lr.fit(training_data)

6. GraphX

6.1 Descripción

GraphX es la librería para procesamiento de grafos de Spark, disponible principalmente en Scala. Permite representar grafos mediante colecciones distribuidas de vértices y aristas y ejecutar algoritmos de análisis de grafos a escala.

6.2 Capacidades

  • Algoritmos clásicos: PageRank, Connected Components, Triangle Counting.
  • Análisis de redes sociales y grafos de relaciones.
  • Transformaciones sobre grafos basadas en operaciones distribuidas.

6.3 Ejemplo conceptual (Scala)

val graph = Graph(vertices, edges)
val ranks = graph.pageRank(0.0001).vertices

7. Integración con ecosistemas externos y conectores

Spark puede integrarse con múltiples tecnologías externas mediante conectores y la Data Sources API. Algunas integraciones comunes incluyen:

  • Hadoop (HDFS) y ecosistema (por ejemplo, Hive).
  • Kafka (ingesta y salida en streaming).
  • Bases NoSQL como Cassandra (mediante conectores).
  • Sistemas de búsqueda como Elasticsearch (mediante conectores).
  • Almacenamiento cloud: AWS S3, Azure Data Lake/Blob, Google Cloud Storage.
  • Acceso a bases de datos relacionales mediante JDBC.

Estas integraciones permiten utilizar Spark como motor de procesamiento sobre datos alojados en múltiples plataformas sin necesidad de moverlos a un sistema único.

8. Comparativa de librerías principales

Librería Propósito Nivel de API Lenguajes

Structured Streaming Procesamiento en tiempo real con API estructurada Alto Scala, Java, Python, R
MLlib Machine Learning distribuido y pipelines Medio/Alto Scala, Java, Python, R
GraphX Procesamiento de grafos Medio Scala

9. Arquitectura unificada del ecosistema

Una característica clave de Spark es que todas las librerías del ecosistema:

  • Comparten el mismo motor de ejecución distribuida.
  • Utilizan el mismo planificador y modelo de tareas.
  • Operan sobre el mismo modelo de particiones.
  • Permiten combinar batch, streaming y ML dentro de un mismo pipeline.

Ejemplo conceptual de pipeline unificado:

  1. Ingesta en streaming (Structured Streaming).
  2. Limpieza y normalización (DataFrames / Spark SQL).
  3. Agregación y preparación de features (Spark SQL / DataFrames).
  4. Entrenamiento o scoring (MLlib).
  5. Persistencia de resultados en almacenamiento distribuido (Parquet, bases de datos, etc.).

10. Conclusión

El ecosistema de librerías de Spark convierte a la plataforma en una solución integral de procesamiento de datos distribuidos. La combinación de Spark Core, Spark SQL, Structured Streaming, MLlib y GraphX permite abordar arquitecturas completas de datos, desde la ingesta hasta el análisis avanzado, compartiendo infraestructura y modelo de ejecución.

En entornos académicos y profesionales, esta unificación facilita el desarrollo y despliegue de soluciones escalables, reduciendo la complejidad tecnológica y permitiendo reutilización de código entre distintos tipos de cargas (batch y streaming) y fases del ciclo de vida del dato.

COMPONENTES PRINCIPALES

Spark SQL / DataFrame API

Permite trabajar con datos tabulares, usar SQL y beneficiarse del optimizador Catalyst y Tungsten para alto rendimiento.

Spark MLlib

Biblioteca de aprendizaje automático con algoritmos escalables (classification, regression, clustering, collaborative filtering).

Spark GraphX

Procesamiento y análisis de grafos (PageRank, connected components, etc.).

Spark Structured Streaming

Procesamiento de streams con abstracción de DataFrame/SQL y garantías de consistencia.

Conectores y paquetes

Conectores para Kafka, JDBC, Cassandra, S3, Elasticsearch y paquetes del ecosistema en spark-packages.org.
