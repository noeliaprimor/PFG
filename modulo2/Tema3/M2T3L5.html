DESCRIPCIÓN

MLlib es la librería de Machine Learning integrada en Apache Spark. Permite entrenar y evaluar modelos de aprendizaje automático sobre grandes volúmenes de datos distribuidos, aprovechando el motor de ejecución de Spark y su capacidad de paralelización.

A diferencia de librerías tradicionales que operan en memoria local (por ejemplo, sobre datasets pequeños), MLlib está diseñada para trabajar en entornos Big Data, donde los datos se distribuyen en múltiples nodos del clúster.

Evolución de MLlib

Históricamente, MLlib ofrecía una API basada en RDD. Sin embargo, en versiones modernas de Spark, el enfoque recomendado es utilizar la API basada en DataFrames dentro del paquete pyspark.ml (o org.apache.spark.ml en Scala).

Esta API estructurada se integra con Spark SQL y permite utilizar:

  • Optimización automática (Catalyst).
  • Pipelines declarativos.
  • Transformers y Estimators reutilizables.

CONTENIDO

Componentes principales de MLlib

Transformer

Un Transformer es un componente que transforma un DataFrame en otro DataFrame mediante el método transform(). Ejemplo: un modelo entrenado o una normalización.

Estimator

Un Estimator es un algoritmo que se ajusta a datos de entrenamiento mediante el método fit(), generando un modelo (que es un Transformer).

Pipeline

Un Pipeline encadena múltiples etapas (stages) de transformación y entrenamiento, permitiendo definir flujos completos de preparación y modelado.

Flujo típico de Machine Learning en Spark

  1. Ingesta y limpieza de datos (DataFrame).
  2. Preparación de características (feature engineering).
  3. División en entrenamiento y test.
  4. Entrenamiento del modelo.
  5. Evaluación.
  6. Persistencia del modelo.

Preparación de datos (Feature Engineering)

VectorAssembler

MLlib requiere que las variables de entrada estén en una única columna de tipo vector. 

from pyspark.ml.feature import VectorAssembler

assembler = VectorAssembler(
    inputCols=["edad", "ingresos"],
    outputCol="features"
)

df_features = assembler.transform(df)
StringIndexer

Convierte variables categóricas en índices numéricos. 

from pyspark.ml.feature import StringIndexer

indexer = StringIndexer(inputCol="categoria", outputCol="categoria_index")
df_indexed = indexer.fit(df).transform(df)
StandardScaler

Normaliza variables numéricas para mejorar el rendimiento de ciertos modelos. 

from pyspark.ml.feature import StandardScaler

scaler = StandardScaler(inputCol="features", outputCol="scaledFeatures")
model_scaler = scaler.fit(df_features)
df_scaled = model_scaler.transform(df_features)

Modelos supervisados

Regresión logística 
from pyspark.ml.classification import LogisticRegression

lr = LogisticRegression(featuresCol="features", labelCol="label")
model = lr.fit(training_data)

predictions = model.transform(test_data)
Árboles de decisión 
from pyspark.ml.classification import DecisionTreeClassifier

dt = DecisionTreeClassifier(featuresCol="features", labelCol="label")
model_dt = dt.fit(training_data)
Random Forest 
from pyspark.ml.classification import RandomForestClassifier

rf = RandomForestClassifier(featuresCol="features", labelCol="label")
model_rf = rf.fit(training_data)

Modelos no supervisados

K-Means (Clustering) 
from pyspark.ml.clustering import KMeans

kmeans = KMeans(featuresCol="features", k=3)
model_kmeans = kmeans.fit(df_features)

clusters = model_kmeans.transform(df_features)

Evaluación de modelos

Evaluación para clasificación 
from pyspark.ml.evaluation import BinaryClassificationEvaluator

evaluator = BinaryClassificationEvaluator(labelCol="label")
auc = evaluator.evaluate(predictions)
Evaluación para regresión 
from pyspark.ml.evaluation import RegressionEvaluator

evaluator = RegressionEvaluator(labelCol="label", metricName="rmse")
rmse = evaluator.evaluate(predictions)

Validación cruzada y ajuste de hiperparámetros

from pyspark.ml.tuning import CrossValidator, ParamGridBuilder

paramGrid = (ParamGridBuilder()
    .addGrid(lr.regParam, [0.01, 0.1])
    .addGrid(lr.elasticNetParam, [0.0, 0.5])
    .build())

crossval = CrossValidator(
    estimator=lr,
    estimatorParamMaps=paramGrid,
    evaluator=evaluator,
    numFolds=3)

cvModel = crossval.fit(training_data)

Pipelines completos

from pyspark.ml import Pipeline

pipeline = Pipeline(stages=[indexer, assembler, lr])
model_pipeline = pipeline.fit(training_data)

predictions = model_pipeline.transform(test_data)

Los pipelines permiten encapsular todo el flujo de preparación y entrenamiento en un único objeto reproducible.

Persistencia del modelo

model_pipeline.write().overwrite().save("/models/modelo_ml")
loaded_model = PipelineModel.load("/models/modelo_ml")

Ventajas de MLlib en entornos Big Data

  • Escalabilidad horizontal.
  • Integración directa con ETL en Spark.
  • Pipelines reproducibles.
  • Compatibilidad con batch y streaming estructurado.

Limitaciones

  • Menor variedad de algoritmos comparado con librerías especializadas (por ejemplo, scikit-learn).
  • Menor flexibilidad en modelos muy personalizados.
  • No sustituye completamente frameworks avanzados de deep learning.

CONCLUSIÓN

MLlib permite integrar Machine Learning dentro del ecosistema Spark de forma escalable y coherente con el resto del pipeline de datos. Su integración con DataFrames y Spark SQL facilita la construcción de modelos reproducibles y optimizados en entornos Big Data.

En arquitecturas modernas de datos, MLlib es una herramienta clave cuando se requiere entrenar modelos sobre grandes volúmenes distribuidos, especialmente en contextos donde los datos ya están siendo procesados con Spark.

EJERCICIO PROPUESTO

1 Objetivo

Implementar un pipeline completo de clasificación binaria utilizando MLlib, incluyendo preparación de datos, entrenamiento, evaluación y ajuste de hiperparámetros.

2 Enunciado

  1. Cargar un dataset estructurado (por ejemplo, CSV o Parquet).
  2. Aplicar StringIndexer a variables categóricas.
  3. Construir la columna "features" con VectorAssembler.
  4. Dividir el dataset en entrenamiento y test.
  5. Entrenar un modelo de regresión logística.
  6. Evaluar el modelo con AUC o accuracy.
  7. Aplicar validación cruzada con diferentes hiperparámetros.
  8. Guardar el modelo entrenado.
