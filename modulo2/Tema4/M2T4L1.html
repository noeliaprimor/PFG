Monitorización y configuración de Spark

  • Descripción
  • Pasos

DESCRIPCIÓN

Aprenderás a monitorizar aplicaciones Spark, interpretar el Spark UI, exportar métricas a sistemas externos (Prometheus/Grafana), configurar parámetros de memoria y CPU, y recoger logs para diagnóstico.

SPARK UI

El Spark UI (por defecto en http://[driver]:4040 mientras la app está viva) muestra:

  • Jobs / Stages / Tasks (tiempos, fallos, shuffle)
  • Storage (RDD/DataFrame cached)
  • Environment (configuración de Spark)
  • Executors (uso de CPU/memoria por executor)

Usa el UI para identificar cuellos de botella: tareas lentas, imbalance de particiones o problemas de GC.

SISTEMA DE MÉTRICAS: Prometheus + Grafana

Spark puede exportar métricas via Dropwizard -> Prometheus collector. Con Grafana puedes crear dashboards de latencia, throughput y utilización.

# ejemplo de configuración en spark-defaults.conf
spark.metrics.conf.*.sink.prometheusServlet.class=org.apache.spark.metrics.sink.PrometheusServlet
spark.metrics.conf.*.sink.prometheusServlet.path=/metrics

CONFIGURACIÓN CLAVE

  • spark.driver.memory y spark.executor.memory — para ajustar memoria del driver y executors.
  • spark.executor.cores — núcleos por executor.
  • spark.sql.shuffle.partitions — particiones por defecto tras operaciones de shuffle.
  • spark.memory.fraction / spark.memory.storageFraction — control de memoria para ejecución vs cache.

Ejemplo: para un cluster con 8 cores por nodo y 32GB RAM, una configuración típica por executor: --executor-memory 8G --executor-cores 4, y ajustar parallelism a un múltiplo de los cores totales.

COMANDOS ÚTILES

# arrancar master y worker (standalone)
$SPARK_HOME/sbin/start-master.sh
$SPARK_HOME/sbin/start-worker.sh spark://$(hostname -f):7077

# ver logs (driver / executor)
tail -f $SPARK_HOME/logs/*driver*.out

# listar aplicaciones en YARN
yarn application -list

EJERCICIO PROPUESTO

Despliega una aplicación de ejemplo y monitoriza su ejecución con el Spark UI. Extrae métricas básicas (tiempo total, tiempo por stage) y documenta las configuraciones usadas.
