DESCRIPCIÓN

Repasaremos cómo securizar un cluster Spark (autenticación con Kerberos, cifrado de comunicaciones con TLS, control de acceso), y estrategias de despliegue (YARN, Kubernetes, docker).

SEGURIDAD

Kerberos

Integración de Spark con Kerberos para autenticación: configurar principal y keytab, y arrancar procesos con kinit.

TLS / SSL

Habilitar TLS para comunicaciones entre driver y executors y para el REST API si es necesario.

# ejemplo: arrancar con keytab
kinit -kt /etc/security/keytabs/spark.service.keytab spark/hostname@REALM
spark-submit --principal user@REALM --keytab /path/to/user.keytab ...

ESTRATEGIAS DE DESPLIEGUE

  • YARN: integración con Hadoop y gestión centralizada de recursos.
  • Kubernetes: despliegue moderno con containers y escalado automático.
  • Standalone: útil para pruebas y entornos pequeños.

Ejemplo de despliegue en Kubernetes con spark-operator o spark-submit con el driver en cluster mode.

EJERCICIO

Investiga cómo configurar TLS para el Spark UI y documenta los pasos necesarios para habilitarlo en un cluster de prueba.
