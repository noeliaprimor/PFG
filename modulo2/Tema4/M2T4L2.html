M2T4L2 — Seguridad y despliegue

Objetivos

  • Entender autenticación (Kerberos), cifrado TLS y control de acceso en clústeres Spark.
  • Aprender buenas prácticas para empaquetar y desplegar aplicaciones (jar, wheel, Docker).

Autenticación y autorización

En entornos corporativos se recomienda integrar Spark con Kerberos y habilitar TLS para comunicaciones seguras entre componentes. Configura spark.network.crypto.enabled y parámetros relacionados cuando sea necesario.

Despliegue y empaquetado

Empaqueta dependencias con --packages, jars o construye una imagen Docker que contenga tu aplicación y las dependencias necesarias.

# ejemplo docker build
Dockerfile:
FROM openjdk:11-jre
COPY my_app.jar /app/
CMD ["spark-submit","--class","com.example.Main","/app/my_app.jar"]
