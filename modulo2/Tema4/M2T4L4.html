DESCRIPCIÓN

Esta lección agrupa recursos para operar aplicaciones Spark: monitorización, configuración, ajuste de rendimiento, seguridad, despliegue, análisis de logs y resolución de errores.

Los enlaces están pensados para pasar de ejecutar notebooks o jobs locales a administrar aplicaciones Spark de forma sistemática, con métricas, evidencias y criterios de diagnóstico.

CONTENIDO

Monitorización de Spark

  • Spark Monitoring: Spark UI, métricas, REST API, logs, event logs y seguimiento de aplicaciones.
  • Spark Web UI: interpretación de jobs, stages, tasks, storage, environment y SQL plans.
  • Job Scheduling: colas, pools, fair scheduler y aislamiento de cargas.

Configuración y rendimiento

  • Spark Configuration: propiedades de memoria, cores, ejecutores, red, shuffle y SQL.
  • Spark Tuning: serialización, uso de memoria, caché, GC y reducción de costes de shuffle.
  • SQL Performance Tuning: AQE, broadcast joins, particionado, coalescing y optimizaciones del motor SQL.
  • Running Spark on Kubernetes: configuración de recursos, imágenes, volúmenes y despliegue en contenedores.

Seguridad y despliegue

  • Spark Security: autenticación, cifrado, ACLs, SSL y recomendaciones de seguridad.
  • Running Spark on YARN: despliegue en entornos Hadoop, colas y configuración de recursos.
  • Spark Standalone Mode: operación de master, workers, alta disponibilidad y recuperación.
  • Docker Compose Documentation: laboratorios reproducibles para validar configuraciones y escenarios de error.

Logs, métricas externas y observabilidad

  • Prometheus Overview: base conceptual para recoger series temporales de métricas.
  • Grafana Documentation: creación de paneles y visualización de métricas operativas.
  • Elastic Documentation: análisis centralizado de logs con Elasticsearch, Logstash, Beats y Kibana.
  • OpenTelemetry Documentation: referencia para trazas, métricas y logs en sistemas distribuidos.

Diagnóstico de errores frecuentes

  • Debugging Spark Applications: pautas para investigar errores en aplicaciones Spark.
  • Configuration Reference: recurso de apoyo para localizar parámetros relacionados con fallos de memoria, red o shuffle.
  • SQL Migration Guide: útil cuando un error aparece tras cambiar de versión de Spark.

RUTA RECOMENDADA

  1. Ejecutar un job y analizar Spark UI antes de cambiar configuración.
  2. Identificar cuellos de botella: CPU, memoria, shuffle, skew, entrada/salida o red.
  3. Ajustar recursos de driver y executors con una hipótesis concreta.
  4. Validar el efecto con métricas, no solo con tiempo total de ejecución.
  5. Documentar configuración, versión, tamaño de datos, logs relevantes y resultado.
