DESCRIPCIÓN

Este tema presenta las bases del procesamiento en tiempo real: ingestión continua de eventos, procesamiento con baja latencia y entrega de resultados. Compararemos modelos batch vs stream y veremos requisitos de disponibilidad, latencia y escalabilidad.

CONCEPTOS CLAVE

  • Latencia: tiempo desde la llegada del evento hasta el resultado.
  • Throughput: tasa de eventos procesados por segundo.
  • Tolerancia a fallos: reintentos, checkpoints y retención de estado.
  • Orden de eventos: ventanas, watermarking y gestión de datos fuera de orden.

EJEMPLO CONSTRUCTIVO (PySpark - Structured Streaming)

from pyspark.sql import SparkSession
from pyspark.sql.functions import explode, split
spark = SparkSession.builder.appName('m3t1l1').getOrCreate()
lines = spark.readStream.format('socket').option('host','localhost').option('port',9999).load()
words = lines.select(explode(split(lines.value, '\\s+')).alias('word'))
counts = words.groupBy('word').count()
query = counts.writeStream.format('console').outputMode('complete').start()
