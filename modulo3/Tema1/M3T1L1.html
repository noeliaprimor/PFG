Introducción a procesamiento en tiempo real

INTRODUCCIÓN

El procesamiento en tiempo real en Apache Spark permite analizar y transformar datos a medida que se generan, reduciendo la latencia entre la llegada del evento y la obtención de resultados. En el contexto Big Data, esta capacidad resulta esencial para casos como detección de fraude, monitorización de sistemas, análisis de logs o procesamiento de eventos IoT.

Spark incorpora capacidades de streaming mediante dos enfoques principales:

  • Spark Streaming (basado en DStreams).
  • Structured Streaming (basado en DataFrames/Datasets).

Actualmente, Structured Streaming es el modelo recomendado por su integración con Spark SQL y su optimización automática mediante Catalyst.

CONTENIDO

Conceptos clave del streaming en Spark

Evento: Unidad de información generada por un sistema externo (por ejemplo, una transacción, un clic web o una lectura de sensor) o como en el ejercicio del Modulo 2, tema 4 por una API de obtención de información de precio de criptomonedas.

Flujo de datos (Stream): Secuencia continua y potencialmente infinita de eventos.

Micro-batch: Modelo en el cual los datos se agrupan en pequeños intervalos temporales (por ejemplo, cada pocos segundos) y se procesan como mini-lotes.

Procesamiento incremental: Cada nueva llegada de datos actualiza los resultados previos sin recalcular el dataset completo.

Evolución del streaming en Spark

Spark Streaming (DStreams)

Fue la primera implementación de streaming en Spark. Se basa en la abstracción DStream, que representa una secuencia de RDDs generados periódicamente.

Aunque funcional, presenta limitaciones en optimización y en integración con Spark SQL.

Structured Streaming

Structured Streaming introduce un modelo declarativo donde el flujo de datos se trata como una tabla infinita. El desarrollador define una consulta utilizando DataFrames, y Spark ejecuta dicha consulta de forma incremental.

Arquitectura típica de streaming con Spark

  1. Fuente de datos (Kafka, archivos, sockets, etc.).
  2. Ingesta mediante readStream.
  3. Transformaciones (filter, groupBy, window, join).
  4. Escritura con writeStream hacia un sink (console, Parquet, base de datos).

Ejemplo básico en Structured Streaming (PySpark)

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("StreamingExample").getOrCreate()

stream_df = (spark
    .readStream
    .format("socket")
    .option("host", "localhost")
    .option("port", 9999)
    .load())

result = stream_df.groupBy("value").count()

query = (result
    .writeStream
    .outputMode("complete")
    .format("console")
    .start())

query.awaitTermination()

Modos de salida

  • Append: solo nuevas filas.
  • Update: filas actualizadas.
  • Complete: resultado completo en cada trigger.

Gestión de ventanas temporales

from pyspark.sql.functions import window

windowed = (stream_df
    .groupBy(window(stream_df.timestamp, "10 minutes"))
    .count())

Las ventanas permiten agrupar eventos por intervalos temporales, fundamentales en análisis de series temporales.

Watermarking y datos tardíos

En escenarios reales, algunos eventos pueden llegar con retraso. Structured Streaming permite definir un watermark para establecer cuánto tiempo se aceptan eventos tardíos. 

stream_df.withWatermark("timestamp", "5 minutes")

Garantías de procesamiento

  • At-least-once: el evento puede procesarse más de una vez.
  • Exactly-once (según fuente y sink): cada evento se procesa exactamente una vez.

Ventajas del streaming en Spark

  • API unificada para batch y streaming.
  • Optimización automática.
  • Escalabilidad distribuida.
  • Integración con ecosistema Big Data.

Retos técnicos

  • Gestión eficiente del estado.
  • Control de latencia.
  • Dimensionamiento correcto de recursos.
  • Monitorización continua.
CONCLUSIÓN

El procesamiento en tiempo real en Apache Spark permite construir aplicaciones analíticas con baja latencia y alta escalabilidad. Structured Streaming ofrece un modelo declarativo que simplifica el desarrollo y garantiza coherencia con el resto del ecosistema Spark.

En arquitecturas modernas de datos, Spark se posiciona como una solución robusta para integrar batch y streaming dentro de un mismo entorno distribuido.
