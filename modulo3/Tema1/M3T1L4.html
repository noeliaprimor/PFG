INTRODUCCIÓN

El procesamiento de datos en tiempo real se ha convertido en un componente esencial en arquitecturas modernas de Big Data. Apache Spark incorpora capacidades de streaming que permiten procesar datos de forma continua y distribuida, reduciendo la latencia entre la generación del evento y la obtención de resultados analíticos.

Dentro del ecosistema Spark existen dos aproximaciones principales al streaming:

  • Spark Streaming (modelo basado en DStreams).
  • Structured Streaming (modelo basado en DataFrames/Datasets).

Structured Streaming representa la evolución natural del modelo hacia un enfoque declarativo, optimizado e integrado con Spark SQL.

CONTENIDO

Spark Streaming (modelo DStream)

Concepto

Spark Streaming fue la primera implementación de procesamiento en tiempo real en Spark. Se basa en la abstracción denominada DStream (Discretized Stream), que representa una secuencia de RDDs generados periódicamente.

El modelo operativo se basa en micro-batches: los datos que llegan en un intervalo de tiempo determinado (por ejemplo, cada 5 segundos) se agrupan y procesan como un pequeño lote.

Flujo de ejecución
  1. Recepción de datos desde una fuente (Kafka, socket, archivos).
  2. División en micro-batches.
  3. Procesamiento distribuido mediante RDD.
  4. Escritura del resultado.
Limitaciones
  • Basado en RDD (menor integración con Spark SQL).
  • API menos declarativa.
  • Optimización limitada frente a DataFrames.

Structured Streaming

Concepto

Structured Streaming es la evolución moderna del streaming en Spark. Trata el flujo de datos como una tabla infinita sobre la cual se pueden ejecutar consultas declarativas usando la API de DataFrames o SQL.

El desarrollador define una consulta, y Spark se encarga de ejecutarla de forma incremental sobre los nuevos datos que van llegando.

Características principales
  • API unificada para batch y streaming.
  • Optimización automática mediante Catalyst.
  • Soporte para ventanas temporales.
  • Gestión de estado y datos tardíos (watermark).
Modelo de ejecución

Structured Streaming funciona principalmente en modo micro-batch, aunque internamente aplica procesamiento incremental y mantiene el estado cuando es necesario.

Ejemplo básico en Structured Streaming (PySpark)

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("StreamingExample").getOrCreate()

stream_df = (spark
    .readStream
    .format("socket")
    .option("host", "localhost")
    .option("port", 9999)
    .load())

result = stream_df.groupBy("value").count()

query = (result
    .writeStream
    .outputMode("complete")
    .format("console")
    .start())

query.awaitTermination()

Modos de salida

  • Append: solo nuevas filas.
  • Update: filas actualizadas.
  • Complete: resultado completo en cada trigger.

Ventanas temporales

Permiten agrupar eventos según intervalos de tiempo. 

from pyspark.sql.functions import window

windowed = (stream_df
    .groupBy(window(stream_df.timestamp, "10 minutes"))
    .count())

Gestión de datos tardíos (Watermarking)

En sistemas reales, algunos eventos pueden llegar con retraso. El watermark define cuánto tiempo se aceptan datos tardíos antes de cerrar una ventana. 

stream_df.withWatermark("timestamp", "5 minutes")

Comparativa general

Aspecto Spark Streaming Structured Streaming
Modelo base DStreams (RDD) DataFrames/Datasets
Optimización automática No Sí
Integración SQL Limitada Total
Recomendación actual Legacy Recomendada

Casos de uso habituales

  • Procesamiento de logs en tiempo real.
  • Detección de fraude.
  • Monitorización de sistemas.
  • Procesamiento de eventos IoT.
  • Integración con Kafka.

CONCLUSIÓN

Spark Streaming introdujo el procesamiento distribuido en tiempo real en el ecosistema Spark, mientras que Structured Streaming lo modernizó mediante un modelo declarativo completamente integrado con Spark SQL.

En arquitecturas actuales de datos, Structured Streaming es la opción recomendada, permitiendo construir pipelines escalables, mantenibles y coherentes con el resto del ecosistema Spark.
