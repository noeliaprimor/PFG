DESCRIPCIÓN

Esta lección recopila recursos para profundizar en arquitecturas de datos en tiempo real: procesamiento de eventos, patrones Lambda y Kappa, Kafka, Spark Streaming, tolerancia a fallos, consistencia, latencia y diseño de pipelines orientados a eventos.

Los enlaces ayudan a conectar la teoría del tema con decisiones prácticas: cuándo mantener una capa batch, cuándo apostar por un log de eventos, cómo reprocesar datos y cómo diseñar pipelines auditables.

CONTENIDO

Arquitecturas Lambda y Kappa

  • Big Data: Principles and best practices: libro de referencia sobre arquitectura Lambda, batch layer, serving layer y speed layer.
  • Designing Data-Intensive Applications: lectura clave para entender logs, replicación, consistencia, stream processing y sistemas distribuidos.
  • Streaming Systems: referencia sobre tiempo de evento, watermarks, ventanas, triggers y modelos de streaming.

Kafka como log de eventos

  • Apache Kafka Introduction: visión general de Kafka como plataforma distribuida de eventos.
  • Apache Kafka Documentation: documentación de topics, particiones, productores, consumidores, retención, seguridad y operación.
  • Kafka Design: explicación del diseño interno, log, replicación, consumidores y garantías.
  • Schema Registry: gestión de contratos de datos y compatibilidad de esquemas en pipelines de eventos.

Spark Streaming y procesamiento incremental

  • Structured Streaming Programming Guide: guía oficial para procesamiento incremental con DataFrames y Datasets.
  • Structured Streaming + Kafka Integration: lectura y escritura de topics Kafka desde Spark.
  • Watermarking: control de eventos tardíos y limpieza de estado.

CDC, event sourcing y captura de cambios

  • Debezium Documentation: captura de cambios de bases de datos como eventos.
  • Debezium Architecture: arquitectura habitual con Kafka Connect y conectores CDC.
  • Event Sourcing: patrón de diseño basado en almacenar cambios como secuencia de eventos.

Diseño, operación y buenas prácticas

  • Kafka Connect: integración de Kafka con bases de datos, sistemas de ficheros, almacenes cloud y buscadores.
  • Docker Compose Documentation: herramienta útil para reproducir laboratorios con Kafka, Spark y servicios auxiliares.
  • Spark Monitoring: monitorización de aplicaciones Spark, logs y métricas.
  • Prometheus Overview: base para monitorizar throughput, latencia y disponibilidad de pipelines.

RUTA RECOMENDADA

  1. Leer primero las diferencias entre procesamiento batch, microbatch y streaming.
  2. Comparar Lambda y Kappa en términos de complejidad, reprocesamiento y latencia.
  3. Revisar Kafka como log de eventos antes de diseñar consumidores.
  4. Estudiar Structured Streaming para conectar tiempo de evento, estado y checkpoints.
  5. Analizar CDC y Schema Registry cuando el origen sean bases de datos o contratos de eventos.

APLICACIÓN AL TEMA

Para consolidar el tema, elabora una ficha comparando Lambda y Kappa con estos criterios:

  • Capas de la arquitectura y responsabilidades.
  • Tratamiento de datos históricos y datos en tiempo real.
  • Coste operativo y duplicación de lógica.
  • Estrategia de reprocesamiento.
  • Herramientas candidatas: Kafka, Spark, almacenamiento analítico y conectores.
