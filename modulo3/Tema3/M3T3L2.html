DESCRIPCIÓN

Guía para integrar Spark Structured Streaming con Kafka: lectura, parseo, escritura y gestión de offsets.

CONTENIDO

Capítulo X: Integración de Apache Spark con Apache Kafka

Arquitectura típica

  1. Productores envían eventos a un topic Kafka.
  2. Kafka almacena eventos en particiones distribuidas.
  3. Spark Structured Streaming consume eventos.
  4. Los datos se transforman y agregan.
  5. Resultados se escriben en un sink (Data Lake, base de datos, Kafka, etc.).

Dependencias y lectura

--packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.x.x
stream_df = (spark
    .readStream
    .format("kafka")
    .option("kafka.bootstrap.servers", "localhost:9092")
    .option("subscribe", "mi_topic")
    .option("startingOffsets", "latest")
    .load())

Parseo de JSON

from pyspark.sql.functions import col

parsed_df = stream_df.selectExpr("CAST(value AS STRING)")

Escritura hacia Kafka

query = (df
    .selectExpr("CAST(key AS STRING)", "CAST(value AS STRING)")
    .writeStream
    .format("kafka")
    .option("kafka.bootstrap.servers", "localhost:9092")
    .option("topic", "output_topic")
    .option("checkpointLocation", "/checkpoint")
    .start())

Gestión de offsets y seguridad

.option("kafka.security.protocol", "SASL_SSL")
.option("kafka.sasl.mechanism", "SCRAM-SHA-256")
CONCLUSIÓN

La integración Kafka-Spark es un patrón probado que, bien configurado, permite construir pipelines robustos y escalables.
